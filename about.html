Project overview

Real-time coastal
intelligence for India's shores

OceanGuard is an open-source coastal weather monitoring system tracking wind, temperature, rainfall, and risk levels across 9 major Indian coastal cities — updated every hour, powered by open data APIs and automated AI briefings.

How it was built
01
The problem
India's coastal population of over 250 million has no unified, free, real-time weather risk layer. Fishermen, port authorities, and coastal planners rely on fragmented government portals with no alerting, no trends, and no AI synthesis. OceanGuard was built to close that gap.
02
Data collection
An n8n automation workflow runs every hour, querying the Open-Meteo API for all 9 cities simultaneously. Each reading captures wind speed, temperature, and rainfall. A risk tier (NORMAL → WATCH → WARNING → EMERGENCY) is computed from city-specific thresholds and written to a Google Sheet as the live database.
03
AI briefings
After each data collection run, a second n8n workflow reads the last 7 days of data, constructs a structured prompt, and calls the Groq API (Llama 3.1 8B) to generate a 3–4 sentence intelligence briefing written in the style of a maritime analyst. The briefing is stored in a separate Briefings sheet and surfaced on the dashboard.
04
The frontend
A pure HTML/CSS/JS dashboard reads directly from the public Google Sheets visualization API — no backend server required. The live dashboard, 7-day trend charts, 24-hour wind forecasts (via Open-Meteo forecast endpoint), and AI briefing are all assembled client-side and refresh automatically every 60 seconds.
Tech stack
Open-Meteo
Free weather API. Wind, temperature, rainfall for any lat/lon. No API key required.
Google Sheets
Live database and public API endpoint via the GViz query interface. Zero hosting cost.
n8n
Workflow automation. Two workflows: hourly data collection and AI briefing generation.
Groq + Llama 3.1
Fast inference for AI briefings. llama-3.1-8b-instant generates analyst-quality summaries in under 2 seconds.
Vanilla HTML/JS
No framework, no build step, no server. The entire frontend is a single deployable HTML file.

Risk methodology
Thresholds
Each city has a city-specific wind threshold (11–14 km/h) tuned to its coastline exposure and historical patterns. Threshold crossings trigger WATCH status. Compound conditions — high wind with significant rainfall — escalate to WARNING or EMERGENCY.
Cyclone seasons
Thresholds auto-tighten during cyclone season: June–November for the Bay of Bengal coast (Chennai, Visakhapatnam, Kolkata, Pondicherry) and April–June for the Arabian Sea coast (Mumbai, Goa, Mangalore, Kochi).
Update frequency
Data is collected every hour via scheduled n8n workflow. The dashboard auto-refreshes every 60 seconds. AI briefings are generated after each collection run and stored with a timestamp.
Forecast layer
The 24-hour wind forecast uses Open-Meteo's hourly forecast endpoint, querying predicted wind speed at each city's coordinates. Threshold crossings in the forecast window surface as predictive alerts on the dashboard.
Alert tiers
Tier Condition Recommended action
NORMAL Wind below city threshold, no significant rainfall No action required. Standard maritime operations.
WATCH Wind at or above threshold, or rainfall ≥ 10 mm Monitor closely. Small vessel operators should check forecasts before departure.
WARNING Wind significantly above threshold, or rainfall ≥ 30 mm Avoid offshore operations. Port authorities should issue advisories.
EMERGENCY Extreme wind or rainfall, sustained multi-day pattern Suspend all non-essential maritime activity. Coordinate with coast guard.

Open source & open data

OceanGuard is built entirely on free, open tools. The source code, n8n workflow JSONs, and Google Sheets template are available on GitHub. Fork it for any coastline in the world.
